# Content Quality Checklist

> Checklist de qualidade pre-publicacao para validacao de conteudo.

---

## Uso

Aplicar ANTES de publicar qualquer peca de conteudo. Gate obrigatorio operado pelo Index.

---

## Camada 1: Fundacao (OBRIGATORIA — todo item deve ser YES)

- [ ] **Espinha Dorsal completa**: TESE clara e defendivel?
- [ ] **Espinha Dorsal completa**: MECANISMO especifico e ensinavel?
- [ ] **Espinha Dorsal completa**: PROVA concreta e verificavel?
- [ ] **Espinha Dorsal completa**: DIRECAO acionavel e explicita?
- [ ] **Template Contract**: Todos os campos dentro dos limites min/max?
- [ ] **Template Contract**: Todos os campos obrigatorios preenchidos?
- [ ] **Brand Consistency**: Tom alinhado com brand voice guide?
- [ ] **Brand Consistency**: Vocabulario de marca respeitado?
- [ ] **Brand Consistency**: Posicionamento coerente com valores?
- [ ] **Gramatica**: Sem erros gramaticais ou ortograficos?
- [ ] **Originalidade**: Angulo unico (nao reformulacao generica)?

> Se QUALQUER item da Camada 1 for NO: **REJEITAR**. Devolver para Arc com feedback.

---

## Camada 2: Excelencia (Score 1-5 cada)

| Criterio | Score | Notas |
|----------|:-----:|-------|
| **Especificidade**: Dados, exemplos, nomes concretos? | _/5 | |
| **Acionabilidade**: Leitor pode agir hoje? | _/5 | |
| **Angulo Unico**: Diferencia de conteudo existente? | _/5 | |
| **Engajamento**: Gera reacao (curiosidade, surpresa, identificacao)? | _/5 | |
| **Clareza**: Entendido na primeira leitura? | _/5 | |

**Score medio**: ___/5

> Score >= 3.5: **APROVAR**. Score 2.5-3.4: **APROVAR COM RESSALVAS**. Score < 2.5: **REJEITAR**.

---

## Camada 3: Verificacoes Adicionais (quando aplicavel)

- [ ] **Compliance**: Claims com evidencia? Disclaimers necessarios incluidos?
- [ ] **Visual**: Marca dagua/logo correto? Cores da marca? Fonte legivel?
- [ ] **Links**: Todos os links funcionam? UTMs configurados?
- [ ] **Hashtags**: Quantidade e mix conforme Template Contract?
- [ ] **CTA**: Claro, especifico e alinhado com objetivo da peca?
- [ ] **SEO** (blog): Title tag, meta description, headers otimizados?

---

## Veredicto Final

| Resultado | Acao |
|-----------|------|
| **APROVADO** | Publicar conforme schedule |
| **APROVADO COM RESSALVAS** | Publicar, documentar melhorias para proximo ciclo |
| **REJEITADO** | Devolver para Arc com feedback especifico por item |
