# Brand Voice Compliance Checklist

## Uso
Aplicar em TODA peca de copy para garantir consistencia com a voz da marca. Especialmente critico em: primeira peca de novo writer, novo canal, nova campanha, ou apos mudanca de guidelines.

## Voice Attributes
- [ ] Todos os voice attributes definidos estao presentes
- [ ] Intensidade de cada attribute adequada ao contexto
- [ ] Attributes NAO se contradizem na peca
- [ ] Balance entre attributes esta proporcional
- [ ] "This, Not That" respeitado (nenhum "Not That" presente)

## Vocabulario
- [ ] Zero never-words utilizadas
- [ ] Always-words presentes onde cabem naturalmente
- [ ] Signature expressions usadas (sem excesso)
- [ ] Jargao tecnico adequado ao canal e audiencia
- [ ] Vocabulario consistente ao longo da peca (mesma palavra para mesmo conceito)
- [ ] Nivel de formalidade consistente (nao muda mid-piece)

## Tom
- [ ] Tom adequado para a SITUACAO (celebracao vs crise vs educacao)
- [ ] Tom adequado para o CANAL (email vs ad vs website)
- [ ] Tom adequado para a AUDIENCIA (prospect vs cliente vs parceiro)
- [ ] Eixo Formality no ponto correto
- [ ] Eixo Emotion no ponto correto
- [ ] Eixo Urgency no ponto correto
- [ ] Eixo Proximity no ponto correto

## Consistencia Interna
- [ ] Tom nao muda sem razao ao longo da peca
- [ ] Estilo de pontuacao consistente
- [ ] Capitalizacao consistente
- [ ] Uso de contractions/abreviacoes consistente
- [ ] Forma de tratamento consistente (voce/tu/nos/a gente)
- [ ] Paragrafos mantêm ritmo e estilo similares

## Consistencia Cross-Channel
- [ ] Mesma voz core adaptada para o canal (nao voz diferente)
- [ ] Reconhecivel como mesma marca em qualquer touchpoint
- [ ] Adaptacoes de tom fazem sentido para o canal
- [ ] Chamadas cross-channel soam coerentes entre si

## Arquetipo de Marca
- [ ] Arquetipo definido esta presente no texto
- [ ] Narrativa alinhada com a jornada do arquetipo
- [ ] Emocoes evocadas sao coerentes com o arquetipo
- [ ] Anti-arquetipo NAO esta presente acidentalmente

## Inclusao e Sensibilidade
- [ ] Linguagem inclusiva (genero, etnia, habilidade)
- [ ] Sem estereotipos ou generalizacoes
- [ ] Termos sensiveis verificados para o contexto cultural
- [ ] Tom respeitoso mesmo em copy agressiva
- [ ] Humor (se usado) nao ofende grupos

## Formatacao e Estilo
- [ ] Convenções de numeros seguidas (extenso vs numeral)
- [ ] Convenções de datas seguidas
- [ ] Convenções de moeda seguidas
- [ ] Uso de bold/italic conforme guidelines
- [ ] Uso de emojis conforme guidelines (permitido/proibido/dosagem)
- [ ] Comprimento de frases dentro dos parametros

## Micro-copy (se aplicavel)
- [ ] Botoes seguem padrao de voz
- [ ] Mensagens de erro sao on-brand
- [ ] Empty states sao on-brand
- [ ] Loading states sao on-brand
- [ ] Confirmacoes seguem tom adequado
- [ ] Notificações seguem padrao

## Teste de Coerencia
- [ ] Teste do "remova o logo": sem marca visivel, reconhece-se a voz?
- [ ] Teste do "leia em voz alta": soa como a marca falaria?
- [ ] Teste do "compare com competitor": claramente diferente?
- [ ] Teste do "mostre para alguem de fora": identifica a personalidade?

## Score Minimo para Publicacao
- Voice Attributes: 100% (todos obrigatorios)
- Vocabulario: >= 90%
- Tom: >= 90%
- Consistencia Interna: 100% (todos obrigatorios)
- Consistencia Cross-Channel: >= 85%
- Arquetipo: >= 80%
- Inclusao: 100% (todos obrigatorios)
- Formatacao: >= 90%
- Teste de Coerencia: >= 75% (min 3 de 4 testes)

## Quando Reprovar
A peca DEVE ser rejeitada se:
- Qualquer never-word esta presente
- Tom muda sem razao dentro da peca
- Linguagem nao inclusiva detectada
- Nao passa no teste "remova o logo"
- Voice attributes se contradizem
