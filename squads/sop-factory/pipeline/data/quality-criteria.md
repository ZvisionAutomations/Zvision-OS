# Quality Criteria — SOP Factory

## Critérios por Agente

### Mário Mapeamento — Mapas de Processo

- [ ] Todos os 4 processos mapeados (onboarding, automação, agente IA, qualidade)
- [ ] Cada processo tem gatilho/input claramente definido
- [ ] Cada processo tem output final definido
- [ ] Cada processo tem pelo menos 2 fases
- [ ] Cada fase tem: nome, responsável (por papel), mínimo 2 passos, output
- [ ] Pelo menos 1 ponto de decisão (go/no-go) por processo
- [ ] Ferramentas/plataformas identificadas por passo (ou marcadas como "TBD")
- [ ] Nenhum nome de pessoa como responsável (usar papéis funcionais)
- [ ] Nível de granularidade consistente entre fases de um mesmo processo

### Sofia SOP — Documentos Markdown

- [ ] Frontmatter YAML completo em todos os 4 documentos (título, tipo, versão, data, status, tags)
- [ ] Resumo executivo (objetivo + gatilho + output) no início de cada SOP
- [ ] Tabela de visão geral de fases presente em cada SOP
- [ ] Callouts Obsidian (`> [!WARNING]` ou `> [!TIP]`) em pelo menos 1 ponto crítico por documento
- [ ] Checkboxes (`- [ ]`) em todos os checklists e passos de validação
- [ ] Nível híbrido aplicado: fases críticas com detalhe completo, demais com formato compacto
- [ ] Tom imperativo em todos os passos (sem "deve ser feito", sem passivo)
- [ ] Tabela de histórico de versões ao final de cada documento
- [ ] Nenhum nome de pessoa como responsável (usar papéis funcionais)
- [ ] Sem em dashes (—) no texto corrido — usar ponto, vírgula ou dois-pontos

### Renata Revisão — Relatório de Revisão

- [ ] Cada documento avaliado em todas as 5 dimensões com nota E justificativa
- [ ] Todo critério com nota < 7 tem localização exata + instrução de correção
- [ ] Veredito por documento claramente definido (APROVADO ou REVISÃO NECESSÁRIA)
- [ ] Relatório inclui sumário executivo com contagem aprovados/rejeitados
- [ ] Próximos passos listados ao final do relatório
- [ ] Prefixos "Correção obrigatória:" e "Sugestão:" usados consistentemente
- [ ] Pelo menos 1 ponto forte identificado por documento (mesmo nos rejeitados)

## Critérios do Squad (visão consolidada)

- [ ] 4 arquivos .md gerados em output/sops/
- [ ] Todos os 4 aprovados por Renata Revisão (ou corrigidos e reaprovados)
- [ ] Documentos prontos para copiar para Obsidian (sem dependências externas)
- [ ] Linguagem consistente com tom da Zvision em todos os documentos
