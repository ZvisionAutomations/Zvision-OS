## Selection Criteria
When choosing tools, evaluate:
1. **Team size fit** (too complex for small teams?)
2. **Integration ecosystem** (connects to our stack?)
3. **Learning curve** (onboarding time?)
4. **Cost per user** (scales with team?)
5. **Client compatibility** (can clients access?)
6. **Data portability** (can we export?)

## Standard Configuration
Every project setup should include:
- [ ] Project board with standard columns (Backlog → Done)
- [ ] Standard labels (feature, bug, tech-debt, discovery)
- [ ] Analytics dashboard with core metrics
- [ ] Shared documentation space
- [ ] Communication channel with client
- [ ] Time tracking (if retainer)
