# Design Systems & UX Frameworks Reference

## Brad Frost — Atomic Design
**Source:** "Atomic Design" (2016)
5 levels: Atoms -> Molecules -> Organisms -> Templates -> Pages
- Atoms: basic HTML elements (buttons, inputs, labels)
- Molecules: simple groups (search form = input + button + label)
- Organisms: complex components (header = logo + nav + search)
- Templates: page-level layouts with placeholders
- Pages: specific instances with real content

## Dan Mall — Design System Governance
**Source:** "Design That Scales" (2023)
- Hot Potato process: design and dev pass work back and forth rapidly
- Interface Inventory: audit all existing UI patterns
- Contribution model: how teams contribute to the design system
- Governance: who decides what goes in, versioning, breaking changes

## Nielsen Norman Group — Usability Heuristics
**Source:** Jakob Nielsen (1994)
10 heuristics: Visibility of system status, Match between system and real world, User control and freedom, Consistency and standards, Error prevention, Recognition over recall, Flexibility and efficiency, Aesthetic and minimalist design, Help users recognize/recover from errors, Help and documentation.

## Google — Material Design Principles
- Material as metaphor, Bold graphic intentional, Motion provides meaning
- Design for accessibility, responsive, platform-adaptive
