# Research Onboarding Checklist

Aplicar ao INICIAR um novo projeto de pesquisa.

## Definicao do Projeto
- [ ] Mercado/industria claramente definido
- [ ] Produto/servico especifico identificado
- [ ] Geografias relevantes mapeadas
- [ ] Horizonte temporal definido (1/3/5 anos)

## Perguntas de Pesquisa
- [ ] Pergunta principal documentada
- [ ] Sub-perguntas desdobradas (3-7)
- [ ] Perguntas priorizadas (must-have vs nice-to-have)
- [ ] Criterios de sucesso definidos (o que constitui uma boa resposta?)

## Stakeholders e Audiencia
- [ ] Stakeholders identificados e mapeados
- [ ] Audiencia do output definida (quem vai ler/usar?)
- [ ] Expectativas de formato alinhadas
- [ ] Decision context claro (que decisao sera informada?)

## Escopo e Recursos
- [ ] Depth level definido e acordado (SCAN/ANALYZE/DEEP DIVE/DEFINITIVE)
- [ ] Timeline e deadlines definidos
- [ ] Agentes designados (Sage, Pulse, Hawk, Scope, Horizon)
- [ ] Budget/recursos alocados (se aplicavel)

## Dados Existentes
- [ ] Pesquisas anteriores inventoriadas
- [ ] Dados internos disponiveis mapeados (CRM, analytics, surveys)
- [ ] Gaps de dados identificados
- [ ] Fontes externas pre-identificadas

## Cross-Squad
- [ ] Squads que precisam dos outputs mapeados
- [ ] Formato de entrega por squad definido
- [ ] Dependencias com outros squads identificadas
- [ ] Handoff protocol comunicado

## Kickoff
- [ ] Research brief criado e aprovado
- [ ] Sprint planning realizado (se applicable)
- [ ] Quality standards comunicados (checklists, depth requirements)
- [ ] Cadencia de updates definida
