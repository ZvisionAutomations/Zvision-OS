# Course Completeness Checklist

Use this checklist before moving from production to launch. Every item must be checked before the course goes live.

---

## Curriculum and Structure

- [ ] Course has 3–8 modules
- [ ] Each module has a written outcome statement
- [ ] Each module has 3–6 lessons
- [ ] Each lesson has a written objective (Bloom's action verb)
- [ ] All objectives map to their module outcome
- [ ] All module outcomes map to a course outcome
- [ ] Prerequisites are documented
- [ ] Total course duration is as advertised

---

## Module Completeness (verify per module)

For **each module**, confirm:

- [ ] Module title is clear and descriptive
- [ ] Module outcome is written and measurable
- [ ] All lessons are present and in correct order
- [ ] At least one assessment per module (quiz or exercise)
- [ ] Module workbook or resource exists (if applicable)
- [ ] Module summary or recap exists

---

## Lesson Completeness (verify per lesson)

For **each lesson**, confirm:

- [ ] Lesson title is specific (not generic like "Introduction")
- [ ] Lesson duration labeled in course navigation
- [ ] Lesson has a clear hook or opening
- [ ] Lesson teaches exactly one concept or skill
- [ ] Lesson includes at least one exercise or checkpoint
- [ ] Lesson ends with takeaways or next step

---

## Assessment Completeness

- [ ] Quizzes exist for all lessons (or at minimum all modules)
- [ ] Each quiz has 3–5 questions
- [ ] All quiz questions have explanations for correct/incorrect answers
- [ ] Module exercises are present and clearly prompted
- [ ] Final project brief is written and accessible
- [ ] Rubric exists for any graded project

---

## Platform Configuration

- [ ] Course title and description published
- [ ] Course thumbnail uploaded
- [ ] All lessons uploaded in correct order
- [ ] All videos are playable (test in student view)
- [ ] All downloadable resources are attached and accessible
- [ ] Quizzes are configured on platform
- [ ] Course access type set (open, drip, date-based)
- [ ] Certificate configured (if applicable)

---

## Sales Page

- [ ] Sales page is published and publicly accessible
- [ ] Headline communicates transformation
- [ ] Modules overview is accurate and matches actual course content
- [ ] Pricing is correct and matches platform checkout
- [ ] CTA buttons link to correct checkout page
- [ ] Sales page loads in under 3 seconds
- [ ] Mobile layout reviewed

---

## Platform Configuration Checklist (final)

- [ ] Pricing set and payment methods configured
- [ ] Checkout tested with a test purchase (access granted correctly)
- [ ] Welcome email configured and tested
- [ ] Community or support channel configured
- [ ] Enrollment confirmation email sent correctly
- [ ] Instructor profile complete on platform
