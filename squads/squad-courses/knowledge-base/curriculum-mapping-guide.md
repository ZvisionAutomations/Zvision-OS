# Curriculum Mapping Guide

How to build a coherent, aligned curriculum from outcomes to individual lessons.

---

## The Mapping Hierarchy

```
Course Outcome
  └── Module Outcome
        └── Lesson Objective
              └── Assessment Item
```

Every level must align upward. If an assessment item doesn't connect to a lesson objective, cut it. If a lesson doesn't connect to a module outcome, cut it.

---

## Step 1 — Define Course Outcomes

Write 3–5 course-level outcomes using Bloom's action verbs (Apply level or above).

| # | Outcome Statement | Bloom's Level | Priority |
|---|-------------------|---------------|----------|
| 1 | [Student will be able to...] | Apply | Core |
| 2 | [Student will be able to...] | Analyze | Core |
| 3 | [Student will be able to...] | Create | Extension |

**Rule:** If you can't write a measurable outcome, the topic isn't ready to be taught.

---

## Step 2 — Break Into Modules

Each module maps to one course outcome (or a cluster of related outcomes).

| Module # | Module Title | Maps to Outcome | Duration Estimate |
|----------|--------------|-----------------|-------------------|
| 1 | [Title] | Outcome 1 | [X hours] |
| 2 | [Title] | Outcome 2 | [X hours] |
| 3 | [Title] | Outcome 3 | [X hours] |

---

## Step 3 — Sequence Lessons per Module

Each module breaks into 3–7 lessons. Sequence matters: foundational before applied.

| Lesson # | Lesson Title | Type | Objective | Duration |
|----------|--------------|------|-----------|----------|
| 1.1 | [Title] | Concept | [Objective] | 8 min |
| 1.2 | [Title] | Demo | [Objective] | 12 min |
| 1.3 | [Title] | Practice | [Objective] | 10 min |

**Lesson types:** Concept / Demo / Practice / Case Study / Review / Assessment

---

## Step 4 — Map Assessments

Each lesson or module has at least one assessment item.

| Lesson | Assessment Type | Bloom's Level | Graded? |
|--------|----------------|---------------|---------|
| 1.1 | Quiz (3 questions) | Remember | No |
| 1.3 | Exercise | Apply | No |
| End of Module 1 | Project milestone | Create | Yes |

---

## Step 5 — Prerequisites Chain

Document what students must know before each module.

| Module | Prerequisites |
|--------|--------------|
| Module 1 | None |
| Module 2 | Module 1 complete |
| Module 3 | Module 2 + external: [skill/tool] |

## Alignment Validation Checklist

Run this before finalizing curriculum:

- [ ] Every lesson has a written objective
- [ ] Every objective connects to a module outcome
- [ ] Every module outcome connects to a course outcome
- [ ] Every assessment item maps to a specific objective
- [ ] No "orphan" lessons (taught but not assessed or connected)
- [ ] No "orphan" outcomes (stated but never addressed in lessons)
- [ ] Prerequisites are documented and sequenced correctly
- [ ] Total duration is realistic for the format and audience

---

## Common Curriculum Mistakes

| Mistake | Fix |
|---------|-----|
| Teaching everything you know | Filter: does this serve the outcome? |
| Skipping prerequisites | Document the prerequisite chain explicitly |
| Assessments that don't match objectives | Rewrite using same Bloom's level as objective |
| All lessons at Remember level | Push at least 50% to Apply or higher |
| No visible sequence logic | Add "Why this order?" notes to each module |
