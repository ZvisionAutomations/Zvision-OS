# Presentation Design Principles

Core principles for creating slides that teach, not distract.

---

## The 10 Fundamental Rules

| # | Rule | Why It Matters |
|---|------|---------------|
| 1 | One idea per slide | Reduces cognitive load, forces clarity |
| 2 | Title = the point, not the topic | "Revenue grew 40%" beats "Revenue" |
| 3 | Max 6 words on a text slide | Audience reads instead of listening |
| 4 | Every visual earns its place | Decoration competes with content |
| 5 | Contrast drives hierarchy | High contrast = important |
| 6 | Consistent layout = less thinking | Familiarity lets focus stay on content |
| 7 | Speaker notes are for the speaker | Don't read slides aloud |
| 8 | Animation only for sequence | Use only when order matters |
| 9 | Dark background for video, light for print | Optimize for medium |
| 10 | Leave whitespace | Crowded slides signal unedited thinking |

---

## Visual Hierarchy

Apply in this order:

1. **Size** — larger = more important
2. **Weight** — bold = key term
3. **Color** — one accent color for emphasis only
4. **Position** — top-left gets read first (F-pattern)
5. **Contrast** — dark on light or light on dark, never gray on gray

**Rule of three:** No more than 3 visual levels per slide. If you have 4 levels, split the slide.

---

## Cognitive Load Management

| High Load (avoid) | Low Load (aim for) |
|-------------------|--------------------|
| Wall of text | Diagram + 1 sentence |
| 3 concepts per slide | 1 concept per slide |
| Inconsistent layout | Repeating template |
| Decorative transitions | No transition or simple fade |
| Clip art and stock photos | Clean icons or original screenshots |

**Sweller's CLT:** Working memory handles ~4 chunks at once. Design for that limit.

## Storytelling in Slides

**Story arc for a teaching presentation:**

1. **Problem** — What's broken or unknown?
2. **Stakes** — Why does it matter?
3. **Solution** — The concept or framework
4. **Proof** — Example, case study, or demo
5. **Action** — What does the audience do now?

**For each module:** Open with a problem the student recognizes. Close with their capability to solve it.

## Slide Types and When to Use Each

| Slide Type | Purpose | Layout |
|------------|---------|--------|
| Title slide | Set context, tone | Big headline + subtext |
| Concept slide | Introduce one idea | Headline + visual or 3 bullets max |
| Diagram slide | Show relationships | Diagram only, no text walls |
| Quote slide | Add credibility, pause | Large text, author attribution |
| Example slide | Make abstract concrete | Before/after, case snippet |
| Exercise slide | Trigger active practice | Clear prompt + timer |
| Summary slide | Consolidate learning | 3 bullets max |
| Transition slide | Signal topic change | Section title only |

---

## Speaker Notes Best Practices

- Write notes as sentences, not keywords
- Include transitions ("This connects to the next slide by...")
- Add timing cues ("Pause here for reactions — ~1 min")
- Mark stories vs facts: [STORY] [STAT] [DEMO]
- Never duplicate slide text in notes verbatim

---

## Workshop Slides vs Lecture Slides

| Dimension | Workshop | Lecture |
|-----------|---------|---------|
| Slide density | Low — lots of exercise slides | Medium — concept-heavy |
| Time per slide | Longer (exercises take 5–20 min) | Shorter (2–4 min) |
| Speaker notes | Facilitation cues | Content delivery notes |
| Visuals | Miro/Figma links, activity prompts | Diagrams, frameworks |
| Audience action | Every 20 min | End of session |

---

## Color and Typography

**Color:** 1 brand color + 1 accent + neutral backgrounds. Never more than 3 colors in a deck.

**Typography:**
- Headlines: 36–48pt
- Body/bullets: 24–28pt
- Captions/labels: 16–18pt
- Never below 16pt — rooms are rarely ideal

**Font pairing:** One sans-serif for body, one display for headlines. Use system fonts when in doubt (Inter, DM Sans, Lato).
