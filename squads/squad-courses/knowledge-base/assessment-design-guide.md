# Assessment Design Guide

How to design assessments that measure real learning, not just recall.

---

## Formative vs Summative

| Type | Purpose | When | Graded? | Examples |
|------|---------|------|---------|---------|
| **Formative** | Check understanding, guide learning | During lessons/modules | Usually not | Quiz, reflection prompt, peer share |
| **Summative** | Measure final achievement | End of module or course | Usually yes | Final project, capstone, portfolio |

**Rule:** Formative = for the learner. Summative = for the record. Both require clear criteria.

---

## Question Types Mapped to Bloom's Levels

| Bloom's Level | Appropriate Question Types | Example |
|--------------|---------------------------|---------|
| **Remember** | Multiple choice, true/false, fill-in | "What are the 3 steps of X?" |
| **Understand** | Short answer, matching, paraphrase | "Explain X in your own words." |
| **Apply** | Scenario problem, case exercise | "Given this situation, what would you do?" |
| **Analyze** | Compare/contrast, breakdown, critique | "What's the difference between X and Y?" |
| **Evaluate** | Justify, defend, assess | "Is this approach effective? Why?" |
| **Create** | Build, design, produce | "Create a plan for..." |

**Avoid:** All questions at Remember level. At least 50% should be Apply or above.

---

## Quiz Design Rules

- **3–5 questions per lesson** (formative)
- **1 correct answer** per multiple choice question
- **Plausible distractors** — wrong answers should be common misconceptions
- **Immediate feedback** — explain why wrong answers are wrong
- **No trick questions** — test knowledge, not reading comprehension
- **Randomize order** if possible to prevent pattern-guessing

## Rubric Creation

Use a 4-level rubric for any graded project:

| Criterion | 4 — Excellent | 3 — Proficient | 2 — Developing | 1 — Beginning |
|-----------|--------------|----------------|----------------|---------------|
| [Criterion 1] | [Description] | [Description] | [Description] | [Description] |
| [Criterion 2] | [Description] | [Description] | [Description] | [Description] |
| [Criterion 3] | [Description] | [Description] | [Description] | [Description] |

**Steps to build a rubric:**
1. List 3–5 most important criteria for success
2. Define what "excellent" looks like for each (start here)
3. Define what "beginning" looks like (the minimum)
4. Fill in middle levels
5. Weight criteria if some matter more than others

## Self-Evaluation

Include self-eval at end of each module:

- "Rate your confidence with [skill]: 1–5"
- "What's one thing you're still uncertain about?"
- "What will you apply first?"

**Purpose:** Metacognition builds retention. It also signals to the student what they should review.

---

## Peer Review

Use when the course has a community component:

**Peer review structure:**
1. Student submits work
2. Assigned to 2 peers with rubric
3. Each peer rates + gives 1 strength and 1 improvement
4. Instructor moderates outlier grades

**When to use:** Cohort-based courses, final projects, writing-heavy courses
**When to avoid:** Solo self-paced courses with no community

---

## Final Project Design

Strong capstone projects share these traits:

| Trait | Description |
|-------|-------------|
| Real-world context | Based on student's actual situation, not a fictional case |
| Integration | Requires using skills from multiple modules |
| Artifact output | Produces something usable (plan, prototype, script) |
| Peer-visible | Can be shared for accountability |
| Instructor feedback | At least light-touch review |

**Example prompt structure:**
> "Using what you learned in this course, [verb + task] for [your context]. Your submission should include [components]. Use the rubric to self-assess before submitting."

---

## Assessment Anti-Patterns

| Anti-Pattern | Problem | Fix |
|-------------|---------|-----|
| Quiz as afterthought | Doesn't connect to objectives | Write quiz before writing content |
| Only recall questions | Doesn't build skill | Map to Apply+ levels |
| Rubric without examples | Too subjective | Add one exemplar per rubric level |
| No feedback on wrong answers | Missed learning moment | Write explanation for every distractor |
| No self-eval | Students don't reflect | Add 3-question self-check at module end |
