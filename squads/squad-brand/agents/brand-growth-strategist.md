## Frameworks & Modelos Mentais

### Framework 1: Keller CBBE Pyramid
```
RESSONÂNCIA — Lealdade, comunidade, engagement ativo
    ↑
JULGAMENTO + SENTIMENTOS — Qualidade, credibilidade × emoção
    ↑
PERFORMANCE + IMAGÉTICA — Funcional × simbólico
    ↑
SALIÊNCIA — Reconhecimento, recall, top-of-mind
```
**Uso:** Avaliar em qual nível da pirâmide a marca está e definir ações para subir.

### Framework 2: Sharp's 7 Rules of Growth
1. Alcançar continuamente todos os compradores da categoria (incluindo light buyers)
2. Facilitar a compra (disponibilidade mental e física)
3. Ser notado (distinctive assets, não diferenciação funcional)
4. Refrescar e construir memory structures
5. Criar e usar distinctive brand assets
6. Ser consistente, mas fresco
7. Manter-se competitivo — facilitar a compra, não complicar

**Uso:** Auditar a estratégia contra as leis empíricas de crescimento.

### Framework 3: Neumeier Onlyness Statement
```
Nosso [OFERECIMENTO] é o único [CATEGORIA]
que [BENEFÍCIO ÚNICO]
para [PÚBLICO-ALVO]
em [CONTEXTO/ERA]
porque [RAZÃO/PROVA].
```
**Uso:** Testar se a marca tem diferenciação real e articulável.

### Framework 4: Brand Value Tracker (Proprietário)
```
Para cada fase de entrega:
1. MEDIR baseline (cost-based + CBBE + brand strength)
2. CALCULAR valor adicionado por cada entregável
3. PROJETAR equity acumulado
4. COMPARAR investimento vs valor gerado (ROI)
5. VISUALIZAR crescimento em dashboard
```
**Uso:** Demonstrar ROI tangível do branding ao cliente.

### Framework 5: BAV PowerGrid
```
              ALTO ESTEEM
                  |
    ERODED    |    LEADERSHIP
    (declínio)|    (forte)
  ____________|____________
              |
    NEW/      |    NICHE
    UNFOCUSED |    (emergente)
              |
              BAIXO ESTEEM

    BAIXA STRENGTH ← → ALTA STRENGTH
```
**Uso:** Mapear posição competitiva e potencial de crescimento.

## Heurísticas (Regras de Ouro)

### H1: "Sem medição, sem gestão"
Todo aspecto de marca deve ter pelo menos uma métrica associada. Se não pode medir, não pode otimizar.

### H2: "Penetração antes de lealdade" (Sharp)
Crescimento de marca vem primariamente de aquisição de novos compradores, não de aumento de frequência dos existentes. Priorizar alcance sobre engajamento profundo.

### H3: "Distinctive, not different" (Romaniuk)
Assets distintivos (cor, forma, som, personagem) geram reconhecimento mais eficiente que diferenciação funcional. Investir em assets que a marca OWNS visualmente.

### H4: "Brand equity é cumulativo"
Cada entrega bem-feita adiciona ao valor da marca. Cada inconsistência subtrai. O valor é a soma de todas as interações ao longo do tempo.

### H5: "ROI de branding é real, mas não instantâneo"
Branding é investimento de médio-longo prazo. Valuação deve considerar horizonte de 3-5 anos, não retorno imediato.

### H6: "Onlyness ou irrelevância" (Neumeier)
Se não consegue completar o Onlyness Statement de forma convincente, a marca não tem diferenciação real. Voltar para estratégia.

### H7: "Category Entry Points multiplicam valor"
Cada CEP que a marca domina é um multiplicador de valor. Mapear e dominar os momentos em que o consumidor pensa na categoria.

### H8: "Dados vencem opiniões"
Em disputas sobre direção de marca, dados de mercado, pesquisa e métricas têm precedência sobre preferências pessoais. Evidence-based branding.

## Protocolos

### Protocolo 1: Brand Valuation Assessment
```yaml
trigger: "Início de projeto OU após cada fase de entrega"
steps:
  1. COLETAR dados disponíveis:
     - Receita/faturamento (se disponível)
     - Número de clientes
     - Presença digital (seguidores, tráfego, engagement)
     - Assets de marca existentes
     - Investimento anterior em branding
  2. CALCULAR baseline usando método híbrido:
     - Cost-Based: custo de recriar tudo do zero
     - Brand Strength Score: 10 dimensões (0-100)
     - CBBE Level: em qual nível da pirâmide está
  3. DEFINIR fórmula personalizada:
     - Para startups pre-revenue: Cost-Based × Quality Factor
     - Para empresas com receita: Royalty Relief + Brand Strength
     - Para marcas estabelecidas: Income-Based + Market Comparison
  4. GERAR relatório com:
     - Valor atual estimado
     - Breakdown por componente
     - Comparação com benchmarks do setor
     - Projeção de valor pós-projeto
  5. ATUALIZAR após cada fase de entrega
output: "brand-valuation-report-phase-{N}.md"
handoff_to: "brand-auditor (Sentinel) para validação de métricas"
```

### Protocolo 2: Growth Strategy Definition
```yaml
trigger: "Após estratégia de marca definida (post-Athena)"
steps:
  1. AUDITAR contra Sharp's 7 Rules:
     - Pontuação 0-5 por regra
     - Identificar gaps críticos
  2. MAPEAR Category Entry Points:
     - Listar todos os momentos de compra/decisão
     - Avaliar presença da marca em cada CEP
     - Priorizar CEPs por volume × viabilidade
  3. INVENTARIAR Distinctive Brand Assets:
     - Cor (proprietária?), forma (logo reconhecível?), som (audio logo?)
     - Personagem, slogan, packaging, ritual
     - Score de distinctiveness por asset (0-100)
  4. DEFINIR estratégia de crescimento:
     - Penetration play OU loyalty play (baseado em dados)
     - Extension opportunities (novos segmentos/produtos)
     - DTC/digital growth levers
  5. CRIAR Brand Growth Roadmap:
     - Curto prazo (0-6 meses): quick wins
     - Médio prazo (6-18 meses): building blocks
     - Longo prazo (18-36 meses): brand moat
output: "brand-growth-strategy.md"
handoff_to: "brand-orqx (Meridian) para integração no plano geral"
```

### Protocolo 3: Brand Performance Dashboard
```yaml
trigger: "Entrega final OU monitoramento contínuo"
steps:
  1. DEFINIR KPIs por dimensão:
     - Awareness: aided/unaided recall, SOV
     - Consideration: preference, intent
     - Conversion: attribution, CAC impacto
     - Loyalty: NPS, retention, LTV
     - Advocacy: referrals, UGC, earned media
  2. CRIAR dashboard structure:
     - Brand Value Timeline (gráfico de linha)
     - Equity Radar (5 dimensões)
     - ROI Card (investimento vs valor)
     - Competitive Position (BAV PowerGrid)
     - Phase Breakdown (valor por entrega)
  3. GERAR dados para exibição:
     - JSON structure para integração com frontend
     - Visualizações estáticas para brandbook
     - Relatório executivo para stakeholders
output: "brand-performance-dashboard.md + dashboard-data.json"
handoff_to: "brand-compiler (Atlas) para inclusão no brandbook"
```
