# Sonic Branding Principles — Knowledge Base

> Fundamentos de identidade sonora e branding auditivo.

---

## 1. O Que É Sonic Branding

Sonic branding é o uso estratégico de som para criar reconhecimento e conexão emocional com uma marca. Inclui:
- **Audio logo** — Assinatura sonora (ex: Intel bong, Netflix ta-dum)
- **Sonic DNA** — Padrão melódico/rítmico que permeia toda a marca
- **UI sounds** — Sons de interface (notificações, confirmações)
- **Audio beds** — Música de fundo para conteúdo
- **Voice guidelines** — Tom e estilo de voz humana/sintetizada

---

## 2. Elementos Sonoros

### 2.1 Audio Logo
| Característica | Guideline |
|---------------|-----------|
| Duração | 1-4 segundos (ideal: 2-3s) |
| Notas | 3-7 notas (simplicidade = memorabilidade) |
| Reconhecimento | Deve ser identificável em 1-2 exposições |
| Versões | Curta (1s), padrão (2-3s), estendida (5-8s) |
| Formatos | WAV 48kHz/24bit (master), MP3 320kbps, AAC 256kbps |

### 2.2 Sonic DNA
- **Definição:** Fragmento musical de 3-5 notas que carrega a essência sonora
- **Uso:** Base para todas as peças sonoras da marca
- **Flexibilidade:** Pode ser tocado em diferentes instrumentos, tempos e contextos
- **Consistência:** Intervalo melódico e ritmo core devem ser preservados

### 2.3 UI Sounds
| Som | Duração | Propósito | Característica |
|-----|---------|-----------|----------------|
| Success | 300-600ms | Confirmação positiva | Tom ascendente, major |
| Error | 300-500ms | Erro/falha | Tom descendente, menor tensão |
| Warning | 400-700ms | Atenção necessária | Suspense, sem resolução |
| Notification | 200-500ms | Nova mensagem/alerta | Distinto, não intrusivo |
| Loading complete | 300-500ms | Tarefa finalizada | Resolução satisfatória |
| Click/tap | 50-150ms | Feedback de interação | Sutil, tátil |

### 2.4 Audio Beds
| Mood | Tempo (BPM) | Instrumentação | Uso |
|------|-------------|----------------|-----|
| Inspiracional | 100-120 | Strings, piano, pads | Hero videos, about us |
| Energético | 120-140 | Beats, synths, guitar | Promos, launches |
| Calmo/Trust | 60-80 | Ambient pads, acoustic | Onboarding, support |
| Sofisticado | 80-100 | Jazz elements, minimal | Premium content |
| Tecnológico | 90-110 | Electronic, glitch, minimal | Product demos, tech |

## 4. Especificações Técnicas

### Formato Master
- **Sample rate:** 48kHz (vídeo) ou 44.1kHz (áudio puro)
- **Bit depth:** 24-bit
- **Formato:** WAV (lossless)
- **Loudness:** -14 LUFS (streaming standard)
- **True peak:** < -1 dBTP

### Formatos de Entrega
| Plataforma | Formato | Bitrate | Notas |
|------------|---------|---------|-------|
| Master | WAV 48kHz/24bit | Lossless | Arquivo original |
| Web | MP3 | 320kbps | Compatibilidade universal |
| Mobile | AAC | 256kbps | iOS nativo |
| Streaming | OGG Vorbis | 320kbps | Web alternative |
| Podcast | MP3 | 192kbps | Voz otimizada |

### Loop Requirements
- **Seamless:** Crossfade no loop point (50-200ms)
- **Durações padrão:** 15s, 30s, 60s, 120s
- **Teste:** Mínimo 3 loops consecutivos sem "click" audível

## 5. Acessibilidade Sonora

### Princípios
1. **Nunca depender apenas de som** para comunicar informação crítica
2. **Sempre oferecer alternativa visual** para feedback sonoro
3. **Volume ajustável** pelo usuário
4. **Respeitar configurações do sistema** (silent mode, do not disturb)

### Frequências Acessíveis
- **Evitar:** Frequências acima de 12kHz (perda auditiva por idade)
- **Preferir:** Faixa 200Hz-4kHz para máxima percepção
- **UI sounds:** Manter entre 500Hz-3kHz

### Duração e Repetição
- **Alarmes:** Máximo 3 repetições, depois silenciar
- **Notificações:** Única execução, não repetir
- **Feedback:** Instantâneo (< 100ms de latência)

## 6. Processo de Criação

### Etapas
1. **Brand Analysis** — Mapear personalidade, valores, arquétipo
2. **Sonic Moodboard** — 10-15 referências sonoras de marcas admiradas
3. **Sonic DNA** — Definir fragmento melódico core (3-5 notas)
4. **Audio Logo** — Criar 3 versões a partir do Sonic DNA
5. **UI Sounds** — Derivar sons de interface do DNA
6. **Audio Beds** — Criar moods baseados na paleta sonora
7. **Testing** — Testar em 5+ dispositivos, A/B com público
8. **Documentation** — Regras de uso, formatos, licenciamento

### Critérios de Avaliação
| Critério | Peso | Descrição |
|----------|------|-----------|
| Memorabilidade | 25% | Reconhecível após 2-3 exposições |
| Distinção | 20% | Diferente de concorrentes |
| Flexibilidade | 20% | Funciona em múltiplos contextos |
| Conexão emocional | 15% | Evoca sentimento alinhado à marca |
| Simplicidade | 10% | Fácil de reproduzir/cantarolar |
| Atemporalidade | 10% | Não preso a tendência sonora |

## 8. DON'Ts

- Copiar ou imitar audio logos existentes
- Usar mais de 7 notas no audio logo
- Criar sons UI que são incômodos ou estressantes
- Ignorar teste em dispositivos de baixa qualidade (celular speaker)
- Mudar o audio logo com frequência (mínimo 5 anos de uso)
- Competir com voz humana no volume (audio beds -6dB sob voz)
