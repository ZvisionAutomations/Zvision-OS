# Task: Optimize Page Conversion

## Metadata
- **Agent:** cro-specialist (Convert)
- **Squad:** squad-paidmedia
- **Complexity:** MEDIUM

## Objetivo
Otimizar landing page para maximizar conversao de trafego pago.

## Steps

1. **Audit above-the-fold**
   - Headline: clara, beneficio-driven, message match com ad
   - Subheadline: expande o beneficio ou elimina objecao
   - CTA: visivel, contraste, copy especifico
   - Social proof: logos, numeros, testimonial
   - Hero image/video: relevante, nao generica

2. **Verificar message match**
   - Ad headline → LP headline: mesmo beneficio/promessa?
   - Ad CTA → LP CTA: mesma acao?
   - Keyword (Google) → LP headline: relevancia?
   - Score: Strong match (mesmo texto), Moderate (mesmo tema), Weak (diferente)

3. **Analisar hierarquia de informacao**
   - Secao 1: Value proposition (acima do fold)
   - Secao 2: Social proof (logos, testimonials, stats)
   - Secao 3: Features/benefits (como funciona)
   - Secao 4: Objection handling (FAQ, garantias)
   - Secao 5: CTA repetido
   - Cada secao responde uma pergunta do usuario

4. **Analisar trust elements**
   - Social proof: depoimentos com nome/foto? Logos de clientes? Numeros?
   - Garantias: money-back? Free trial? Cancel anytime?
   - Seguranca: SSL, privacy policy, payment badges
   - Authority: certificacoes, premios, media mentions

5. **Recomendacoes**
   - Quick wins (<1 dia): ajustar headline, CTA copy, social proof placement
   - Medium effort (1 semana): reestruturar page, adicionar secoes
   - High effort (2+ semanas): redesign, novo teste A/B

## Output
LP optimization report com audit, score e recomendacoes priorizadas.

## Quality Gates
- [ ] Message match avaliado
- [ ] Above-the-fold auditado
- [ ] Trust elements verificados
- [ ] Recomendacoes priorizadas por esforco/impacto

## Quando Usar
- Antes de launch de campanha paga
- Quando CVR da LP esta baixo
- Apos launch (otimizacao continua)
