# CRO Audit Checklist

## 1. Above-the-Fold
- [ ] Headline clara e orientada a beneficio (nao feature)
- [ ] Subheadline expande a promessa ou elimina objecao
- [ ] CTA visivel sem scroll
- [ ] CTA com contraste visual alto
- [ ] CTA com copy especifico (nao "Enviar" ou "Clique aqui")
- [ ] Imagem/video relevante (nao stock generico)
- [ ] Social proof presente acima do fold (logos, numeros, ou 1 testimonial)

## 2. Message Match
- [ ] Headline do ad ≈ headline da LP (mesmo beneficio/promessa)
- [ ] CTA do ad = CTA da LP (mesma acao)
- [ ] Visual consistency entre ad e LP
- [ ] Keyword do Google ≈ headline da LP (para Search)
- [ ] Score: Strong / Moderate / Weak

## 3. Trust Elements
- [ ] Testimonials com nome e foto reais
- [ ] Client logos (min 3-5)
- [ ] Stats/numeros concretos ("10.000+ clientes", "98% satisfacao")
- [ ] Garantia explicita (money-back, free trial, cancel anytime)
- [ ] Security badges (SSL, payment logos, privacy)
- [ ] Certificacoes ou premios (se aplicavel)

## 4. Form & CTA
- [ ] Numero de campos minimo para qualificacao
- [ ] Campos required vs optional marcados
- [ ] Inline validation presente
- [ ] Error messages claras e proximas do campo
- [ ] Mobile: input types corretos (email, tel, etc.)
- [ ] Mobile: tap targets >= 48x48px
- [ ] CTA repetido abaixo do fold

## 5. Content Hierarchy
- [ ] Secao 1: Value proposition (acima do fold)
- [ ] Secao 2: Social proof
- [ ] Secao 3: Features/benefits (como funciona)
- [ ] Secao 4: Objection handling (FAQ, garantias)
- [ ] Secao 5: CTA final
- [ ] Hierarquia visual clara (headings, spacing, visual hierarchy)

## 6. Mobile Experience
- [ ] Renderiza corretamente em mobile (320-767px)
- [ ] Texto legivel sem zoom (min 16px)
- [ ] Nao ha scroll horizontal
- [ ] Images responsivas
- [ ] Form usavel com thumb
- [ ] CTA facilmente acessivel

## 7. Performance
- [ ] Lighthouse Performance >80
- [ ] LCP <2.5s (mobile)
- [ ] CLS <0.1
- [ ] Page load <3s em 3G
- [ ] No console errors
