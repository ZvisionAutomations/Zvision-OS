# Client Onboarding Checklist

## Purpose
Checklist de onboarding de novo cliente — garante que cada novo cliente tem uma experiência de ativação consistente, profissional e que acelera time-to-value nos primeiros 30 dias.

## Pre-Requisites
- Contract signed (Closed-Won in CRM)
- Handoff document from sales received
- CSM assigned

---

## Phase 1: Pre-Kickoff (Day 0-3)

### Internal Preparation
- [ ] Review complete discovery notes and proposal
- [ ] Review signed contract scope and terms
- [ ] Assign delivery team members
- [ ] Create account in CRM (CS view)
- [ ] Initialize Health Score baseline
- [ ] Set renewal date reminder (90 days before)
- [ ] Schedule internal onboarding brief with delivery team

### Client Communication
- [ ] Send welcome email (within 24h of contract)
- [ ] Introduce CSM and delivery team leads
- [ ] Share onboarding timeline and next steps
- [ ] Request access credentials (platforms, accounts, assets)
- [ ] Send kickoff meeting invitation (within 5 business days)
- [ ] Share pre-kickoff questionnaire

### Administrative
- [ ] Create project in project management tool
- [ ] Set up shared drive/folder for client
- [ ] Create Slack/Teams channel (if applicable)
- [ ] Add client contacts to relevant communication lists
- [ ] Verify billing setup and first invoice scheduled

---

## Phase 2: Kickoff Meeting (Day 3-7)

### Meeting Preparation
- [ ] Customize kickoff deck with client-specific details
- [ ] Review pre-kickoff questionnaire responses
- [ ] Prepare success plan draft
- [ ] Confirm all stakeholders invited (both sides)

### During Kickoff
- [ ] Introductions (team members and roles)
- [ ] Confirm objectives and expected outcomes
- [ ] Walk through success plan draft
- [ ] Agree on communication cadence
- [ ] Define escalation path (both sides)
- [ ] Review first 30-day milestones
- [ ] Identify quick-win opportunity
- [ ] Confirm next meeting date

### Post-Kickoff
- [ ] Send meeting summary within 24h
- [ ] Share finalized success plan
- [ ] Create action items in project management tool
- [ ] Update CRM with kickoff notes
- [ ] Begin work on first deliverable

---

## Phase 3: First 30 Days (Day 7-30)

### Week 1-2: Foundation
- [ ] All access credentials received and verified
- [ ] Current state audit completed (brand, digital, content, etc.)
- [ ] Quick-win identified and work started
- [ ] First weekly sync completed
- [ ] Client feedback on communication style gathered

### Week 2-3: First Deliverables
- [ ] Quick-win delivered and celebrated
- [ ] Primary workstream initiated
- [ ] First progress report sent
- [ ] Any issues or blockers escalated and resolved
- [ ] Client champion relationship deepening

### Week 3-4: Validation
- [ ] Second weekly sync completed
- [ ] Client satisfaction check (informal)
- [ ] Success plan updated with actual progress
- [ ] Health Score updated with first data points
- [ ] Stakeholder engagement confirmed (beyond champion)

---

## Phase 4: 30-Day Checkpoint

### Review Meeting
- [ ] Schedule 30-day review meeting
- [ ] Prepare progress report vs success plan milestones
- [ ] Document wins achieved (quantified where possible)
- [ ] Identify any gaps or concerns
- [ ] Update Health Score formally

### Assessment
| Check | Status | Notes |
|-------|--------|-------|
| Quick-win delivered | ✅/❌ | |
| Client satisfaction ≥ 8/10 | ✅/❌ | |
| Communication cadence established | ✅/❌ | |
| Health Score ≥ 70 | ✅/❌ | |
| Success plan milestones on track | ✅/❌ | |
| No unresolved issues | ✅/❌ | |
| Champion relationship strong | ✅/❌ | |

### 30-Day Gate Decision
- [ ] **GREEN** — Onboarding successful, transition to regular cadence
- [ ] **YELLOW** — Minor gaps, specific action plan for next 30 days
- [ ] **RED** — Significant issues, escalation required

---

## Handoff Quality Verification

### From Sales (Must Receive)
- [ ] Discovery notes (SPIN/Gap analysis)
- [ ] Proposal that was signed (scope definition)
- [ ] MEDDIC scorecard (stakeholder map, decision process)
- [ ] Key contacts with roles and preferences
- [ ] Promises made during sales process (explicit and implied)
- [ ] Known risks or sensitivities
- [ ] Client's definition of success (in their words)
