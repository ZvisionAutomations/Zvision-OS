# Proposal Quality Checklist

## Purpose
Gate de qualidade obrigatório antes de enviar qualquer proposta comercial. Garante que cada proposta segue o framework Three-Options, conecta com discovery, e maximiza probabilidade de close.

## Pre-Requisites
- Deal qualification checklist passed (MEDDPICC ≥ 10/16)
- Discovery summary completed and reviewed
- Pricing approved by leadership (if required)

---

## Section A: Discovery Connection (5 points)

- [ ] **(1)** Proposal references specific pains from discovery call
  - Client's words used, not our jargon
  - At least 3 specific pain points from discovery mentioned

- [ ] **(1)** Gap analysis included (cost of status quo)
  - Quantified impact of current state
  - "If nothing changes, this costs R$X per year"

- [ ] **(1)** Future state description matches client's vision
  - Uses language the client used in discovery
  - Measurable outcomes tied to their stated goals

- [ ] **(1)** Decision criteria from MEDDIC addressed
  - Each stated decision criterion mapped to our approach
  - No criteria left unaddressed

- [ ] **(1)** Stakeholder concerns pre-addressed
  - Known concerns from different stakeholders addressed proactively
  - Technical, business, and personal concerns considered

## Section B: Structure & Format (6 points)

- [ ] **(1)** Three-options framework used (Good/Better/Best)
  - Three distinct options with clear differentiation
  - Each option solves the core problem (not just more hours)

- [ ] **(1)** Executive summary < 1 page
  - Compelling, outcome-focused opening
  - Could stand alone if decision maker only reads this page

- [ ] **(1)** Clear scope per option
  - "What's INCLUDED" and "What's NOT INCLUDED" explicit
  - No ambiguity about deliverables

- [ ] **(1)** Timeline with milestones
  - Phase-by-phase timeline (not just total duration)
  - Key milestones and check-in points defined

- [ ] **(1)** Team assignment with relevant bios
  - Specific team members named (not "a senior designer")
  - Bios highlight relevant experience (not generic resume)

- [ ] **(1)** Professional formatting
  - No typos, grammar errors, or formatting issues
  - Brand-consistent design
  - Client name and details correct throughout

**Section B Score:** ___ / 6

## Section C: Value Framing (5 points)

- [ ] **(1)** Investment framed against ROI
  - "For R$X investment, you can expect R$Y return"
  - Not just a price list — a business case

- [ ] **(1)** Per-option ROI or value stated
  - Each option has its own expected outcome and ROI estimate
  - Higher options show higher ROI (not just higher cost)

- [ ] **(1)** Pricing logic transparent
  - Value-based (not cost-plus or hourly)
  - Client understands WHY it costs what it costs

- [ ] **(1)** Comparison to cost of inaction included
  - "Doing nothing costs you R$X/year in lost [opportunity/revenue]"
  - Makes the investment feel like a bargain by comparison

- [ ] **(1)** Social proof embedded
  - At least 1 relevant case study (similar industry/problem/size)
  - Specific metrics from the case study (not just "they were happy")

## Section D: Closing Mechanics (5 points)

- [ ] **(1)** Clear next steps defined
  - Specific action: "Sign by [date], kickoff by [date]"
  - Not vague "let us know what you think"

- [ ] **(1)** Urgency element present
  - Timeline driver (market, capacity, season, budget cycle)
  - Genuine, not manufactured

- [ ] **(1)** Terms and conditions included
  - Payment terms clear
  - Contract duration defined
  - Cancellation/change terms stated

- [ ] **(1)** Easy to say yes
  - Signature line or e-sign link
  - Payment method clear
  - No barriers to action

- [ ] **(1)** Follow-up cadence planned
  - When will we follow up after sending
  - What we'll do if no response by [date]
  - Not left to chance

## Three-Options Audit

### Options Balance
| Check | ✓/✗ |
|-------|------|
| Option 2 (Better) is clearly the recommended choice | |
| Option 1 (Good) is viable but limited (creates contrast) | |
| Option 3 (Best) is aspirational but justifiable | |
| Price gap between options is intentional, not arbitrary | |
| Each option adds genuine VALUE (not just scope/hours) | |
| All three options solve the core problem from discovery | |
| Presentation order: Option 3 → Option 1 → Option 2 (recommended) | |
