# Agency Productization Playbook

## Why Productize (The Business Case)

### Custom vs Productized Services
```
CUSTOM SERVICES:
  Revenue model: Time & materials (hourly/daily)
  Pricing: Negotiated per project
  Scope: Defined per engagement
  Delivery: Customized each time
  Margins: 40-55% (variable)
  Scalability: Linear with headcount
  Predictability: Low (scope creep, estimate variance)

PRODUCTIZED SERVICES:
  Revenue model: Fixed price per deliverable
  Pricing: Published or standardized
  Scope: Pre-defined (what's IN and OUT)
  Delivery: Standardized process + templates
  Margins: 60-75% (improving with scale)
  Scalability: Sub-linear with headcount (efficiency gains)
  Predictability: High (known scope, known effort, known outcome)

THE SHIFT:
  Custom services sell TIME → "We'll work on your project for X hours"
  Productized services sell OUTCOMES → "We'll deliver X result in Y timeframe"
```

### Financial Impact
```
BEFORE PRODUCTIZATION:
  Average project: R$50K
  Utilization: 65% (35% in scoping, proposals, rework)
  Effective margin: 45%
  Revenue per employee: R$180K/year
  Time to proposal: 2-3 weeks

AFTER PRODUCTIZATION:
  Average productized service: R$25K (but 3x volume)
  Utilization: 80% (20% in standardized pre-work)
  Effective margin: 65%
  Revenue per employee: R$280K/year
  Time to proposal: 1-2 days (standard pricing)

Key insight: Smaller, standardized engagements with
higher margins often outperform large, custom projects.
```

## Productization Framework

### Step 1: Identify Candidates
```
PRODUCTIZATION READINESS SCORECARD

For each service, score 1-5:

| Factor | Score | Notes |
|--------|-------|-------|
| Repetition frequency (how often do we do this?) | | |
| Process maturity (do we have SOPs?) | | |
| Outcome predictability (can we guarantee results?) | | |
| Client demand (is this a common need?) | | |
| Competitive differentiation (are we notably better?) | | |
| Pricing simplicity (can we set a fixed price?) | | |
| Delivery scalability (can junior team deliver?) | | |
| TOTAL | /35 | |

Threshold:
  28-35: Excellent candidate — productize immediately
  21-27: Good candidate — needs some standardization first
  14-20: Possible — requires significant process work
  <14: Not ready — keep as custom for now
```

### Step 2: Define the Package
```
PRODUCTIZED SERVICE DEFINITION

Service Name: [Catchy, branded name]
  Bad: "Website Redesign Services"
  Good: "Site Sprint — Your New Website in 30 Days"

One-Liner: [What it is in one sentence]
  "A complete website redesign delivered in exactly 30 days,
  guaranteed."

Target Client: [Specific ICP for this service]
  "B2B SaaS companies with 50-500 employees who need to
  refresh their marketing site to support a new positioning."

The Promise: [Specific, measurable outcome]
  "A high-converting marketing website, live in 30 days,
  with a conversion-optimized design based on your ICP."

What's Included:
  ✓ Discovery workshop (1 day)
  ✓ Wireframes for 8 pages
  ✓ Custom design (desktop + mobile)
  ✓ Development and QA
  ✓ Content migration assistance
  ✓ 1 round of revisions per page
  ✓ Launch support

What's NOT Included (Scope Boundaries):
  ✗ Content writing (can be added as separate service)
  ✗ Custom integrations beyond standard (Hubspot, GA, etc.)
  ✗ More than 8 pages (each additional: R$X)
  ✗ Ongoing maintenance (separate retainer available)

Fixed Price: R$40,000
Timeline: 30 calendar days
Guarantee: "If we don't launch by day 30, the overage is on us."
```

### Step 3: Standardize Delivery
```
DELIVERY SOP — [Service Name]

PHASE 1: KICKOFF (Day 1-3)
  Inputs: Signed contract, brand guidelines, access credentials
  Activities:
    □ Kickoff call (1 hour, standard agenda)
    □ Discovery questionnaire sent (standard template)
    □ Content inventory from existing site
  Outputs: Project brief (standard template), timeline confirmed
  Owner: Project Manager
  Template: [link to kickoff deck template]

PHASE 2: DESIGN (Day 4-15)
  Inputs: Project brief, brand guidelines
  Activities:
    □ Wireframes (standard page templates)
    □ Visual design (using design system components)
    □ Client review (1 round, standard feedback form)
  Outputs: Approved designs
  Owner: Lead Designer
  Template: [link to design template]

PHASE 3: BUILD (Day 16-25)
  Inputs: Approved designs
  Activities:
    □ Development (using standard components)
    □ Content migration
    □ QA testing (standard checklist)
  Outputs: Staging site ready for review
  Owner: Lead Developer
  Template: [link to QA checklist]

PHASE 4: LAUNCH (Day 26-30)
  Inputs: Staging site approved
  Activities:
    □ Client final review (standard sign-off form)
    □ DNS/hosting configuration
    □ Launch checklist execution
    □ Post-launch monitoring (48 hours)
  Outputs: Live site
  Owner: Project Manager
  Template: [link to launch checklist]
```

### Step 4: Build the Value Ladder
```
SERVICE PORTFOLIO — PRODUCTIZED TIER MODEL

TIER 1: DIAGNOSTIC (Entry Point)
  Purpose: Low-commitment way to demonstrate value
  Examples:
    "Brand X-Ray" — Brand audit in 5 days (R$5K)
    "Funnel Diagnostic" — Full-funnel analysis (R$8K)
    "SEO Health Check" — Technical + content audit (R$6K)
  Margin: 70%+ (mostly templates and analysis)
  Conversion to Tier 2: Target 40%+

TIER 2: PROJECT (Core Delivery)
  Purpose: Deliver defined outcome with clear scope
  Examples:
    "Site Sprint" — Website in 30 days (R$40K)
    "Content Engine" — 90-day content program (R$25K)
    "Brand Sprint" — Complete brand strategy (R$30K)
  Margin: 60-65%
  Conversion to Tier 3: Target 50%+

TIER 3: PROGRAM (Ongoing Partnership)
  Purpose: Recurring revenue through continuous value
  Examples:
    "Growth Partner" — Monthly full-funnel marketing (R$15K/mo)
    "Brand Guardian" — Ongoing brand management (R$8K/mo)
    "Performance Engine" — Ongoing paid media (R$10K/mo)
  Margin: 55-65%
  NRR Target: 110%+

ASCENSION PATH:
  Diagnostic → proves expertise (delivers insight)
  Project → delivers transformation (delivers result)
  Program → sustains value (delivers growth)
```

## Naming & Packaging Best Practices

### Naming Rules
```
GOOD NAMES:
  ✓ Evoke the outcome, not the process
  ✓ Memorable and distinct
  ✓ Short (2-3 words max)
  ✓ Action-oriented or result-oriented

EXAMPLES:
  "Site Sprint" (speed + website)
  "Brand X-Ray" (diagnostic + brand)
  "Growth Blueprint" (strategy + growth)
  "Content Engine" (ongoing + content)
  "Funnel Forge" (creation + funnel)

ANTI-PATTERNS:
  ✗ "Website Design Package A" (generic, forgettable)
  ✗ "Full-Service Digital Marketing" (too broad)
  ✗ "Premium Social Media Management" (adjective-based)
  ✗ "Custom Strategy Development" (says nothing specific)
```

### Packaging Principles
```
1. EACH PACKAGE SOLVES ONE SPECIFIC PROBLEM
   Not "we do everything" — but "we solve THIS"

2. SCOPE IS CRYSTAL CLEAR
   Client knows exactly what they get (and what they don't)

3. PRICE IS JUSTIFIED BY VALUE, NOT EFFORT
   "R$40K for a website that increases conversions 50%"
   Not "R$40K for 200 hours of design and development"

4. DELIVERY IS PREDICTABLE
   "Delivered in 30 days" — not "estimated 4-8 weeks"

5. GUARANTEE REDUCES RISK
   "Or we work free until it's done" removes buyer hesitation
```

## Metrics for Productized Services

### Per-Service Metrics
| Metric | Target | Why It Matters |
|--------|--------|---------------|
| Units sold per quarter | Growing | Volume/demand signal |
| Average delivery time | Stable or decreasing | Efficiency signal |
| Delivery margin | > 60% | Profitability signal |
| Client satisfaction (CSAT) | > 4.5/5 | Quality signal |
| Upsell rate to next tier | > 30% | Ladder effectiveness |
| Scope change frequency | < 10% | Scope clarity signal |
| Repeat purchase rate | > 25% | Value confirmation |

### Portfolio Metrics
| Metric | Target | Why It Matters |
|--------|--------|---------------|
| % revenue from productized | > 40% | Transition progress |
| Revenue per employee (productized) | > R$250K | Efficiency gains |
| Time from lead to proposal | < 2 days | Speed advantage |
| Win rate on productized | > 35% | Easier to buy |
| Average margin (productized vs custom) | > 15pp higher | Margin improvement |
